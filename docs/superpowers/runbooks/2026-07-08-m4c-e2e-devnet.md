# M4c — human end-to-end devnet runbook

The M4c gate. Execute this yourself against live devnet before M4d. Every step lists its **PASS** condition — if one fails, stop and fix before declaring M4c done.

**Prereqs**
- A Phantom (or Backpack) browser wallet, set to **Devnet**, using a **fresh** account with 0 SOL (proves the whole cold-start loop).
- Node 22 (`nvm use 22`). The app launcher (`.claude/launch.json` → "app") already pins this.
- Repo built once: `pnpm install && pnpm --filter @ansem/sdk build`.

Machine facts (verified 2026-07-08): program `8Q9EnK7ydn6ywo7ZxeqhubqYybf7FFNNwnz8JzJjXZjz`; one-popup entry lands in a single ~739-byte tx (T4 spike); real-browser PDA derivation confirmed.

## 0. Start backend + app

- [ ] **Keeper** (public RPC dodges Helius 429; ER stays MagicBlock; HTTP on 8787):
  ```bash
  source scripts/devnet-env.sh
  export ANCHOR_PROVIDER_URL=https://api.devnet.solana.com WS_ENDPOINT=wss://api.devnet.solana.com
  pnpm run keeper:dev
  ```
  **PASS:** logs show a round `create` + `delegate` (round OPEN), then idle until deadline.

- [ ] **App:** `pnpm run app:dev` (serves on :3100) → open `http://localhost:3100`.
  **PASS:** the bull-head board renders; top-right shows `KEEPER: CONNECTED`; HUD shows the live round # + countdown. No wallet connected yet → no write column.

---

## 1. Connect a fresh wallet

- [ ] Click **Select Wallet** (top-right) → pick your devnet wallet (fresh account).
  **PASS:** the button shows your address; the **write column appears** below the board (ESCROW panel + "Enter round · one popup").
- [ ] Fund it: click **get devnet SOL** (bottom of the write column) → airdrop ~0.2 SOL at the faucet.
  **PASS:** wallet shows a devnet SOL balance (may take a few seconds).

## 2. Deposit → escrow

- [ ] In **ESCROW**, type `0.05` → **Deposit** → approve the wallet popup.
  **PASS:** after confirm, ESCROW reads `0.05 SOL`. (One popup — this is a value move, wallet-signed by design.)

## 3. Enter the round — ONE popup

- [ ] Click **Enter round · one popup** (only enabled while the round is OPEN).
  **PASS:** **exactly ONE** wallet approval appears. After it confirms (a few seconds for delegation to land), the write column swaps the button for the **STAKE · GASLESS** rail.
  - Under the hood this one tx did: `init_miner` (first time) + `createSessionV2` + `join_round` + `delegate_miner`.

## 4. Stake — gasless (ZERO popups)

- [ ] Click a **bull tile** on the board (it gets a white outline = selected) → in STAKE, type `0.02` → **Stake · gasless**.
  **PASS:** **NO wallet popup.** Within a few seconds the tile lights **green** and the pot/HUD tick up live (the keeper's snapshot reflects your stake to every viewer).
- [ ] Select a second tile and stake again.
  **PASS:** still no popups; second tile lights.

## 5. Settle + reveal

- [ ] Wait for the countdown to hit 0. The keeper runs **commit-then-settle** — it undelegates the ER round back to L1 *before* requesting VRF (decide.ts), so watch the keeper logs in this order: `round committed back to L1` → `request_settle posted` → (the MagicBlock VRF oracle fulfills → round SETTLED) → `round swapped -> CLAIMABLE`.
  **PASS:** the board plays the reveal; the winning tile flashes **gold**; state goes SETTLED → CLAIMABLE.

## 6. Claim ANSEM

- [ ] When the write column shows **ROUND #N · WON** with **Claim ANSEM**, click it → approve the popup.
  **PASS:** the claim confirms; your wallet's **ANSEM** token balance is > 0 (add the mint in the wallet if it doesn't auto-show — the mint is the program's `ansem_mint` PDA). The claim panel clears.

## 7. Recovery sanity (optional)

- [ ] If a round ever voids (keeper logs `cancel_round`; state CLOSED), the panel shows **Refund** instead → click → approve.
  **PASS:** escrow SOL is restored and withdrawable again.

## 8. Withdraw (optional)

- [ ] When no round is active (ESCROW not locked), enter an amount → **Withdraw** → approve.
  **PASS:** SOL returns to the wallet; ESCROW decreases.

---

## Result

All PASS = **M4c done**. The full loop — connect → deposit → one-popup entry → gasless stake → real-VRF reveal → claim ANSEM — works in-browser on live devnet. Proceed to **M4d** (reveal polish + Vercel/keeper deploy).

If any step fails, note which PASS condition broke and the keeper/browser console output, then fix before M4d.
